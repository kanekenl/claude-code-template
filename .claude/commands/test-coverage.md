# /test-coverage スキル

<command-name>test-coverage</command-name>

<description>
カバレッジを計測し、重要経路のテスト不足を特定して補強する。
</description>

<usage>
/test-coverage
/test-coverage <対象モジュール/ファイル>
</usage>

<instructions>

## 実行手順

### 1. カバレッジ計測
- プロジェクトの標準手段で coverage を生成する（例: `npm test -- --coverage` など）
- 失敗した場合は `/build-fix` に切り替えて先に通す

### 2. 重点領域を選ぶ
- 仕様に直結する部分（Planの受け入れ条件に対応するコード）
- バグりやすい境界（入力検証、権限、状態遷移、エラー処理）
- ユーザー影響が大きい部分（決済/認証/データ更新など）

### 3. テスト追加の方針
- 実装詳細ではなく “観測可能な振る舞い” をアサートする
- 成功/失敗/例外/境界値を揃える
- 既存のテストパターンに合わせ、無理に網羅を狙わない（重要経路優先）

### 4. 再計測と報告
- 追加後に再度 coverage を取り、改善を確認する
- 目標値（例: 80%）は目安として、重要経路が守れているかを優先する

</instructions>
